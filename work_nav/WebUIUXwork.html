•  Open Graph

: 원활한 SNS 링크 공유를 목표로 대표적인 sns 오픈그래피 적용
- < meta property="og" >적용

•  Favicon

: 5개의 이미지와 1개의 JSON 파일로 파비콘 최적화
- .ico 32x32: 예전 브라우저들

- .svg: 대부분 최신 브라우저들, 다크모드도 지원 - apple-touch-icon 180x180: 애플 기기 - 192x192: 홈스크린 - manifest(JSON): PWA 및 안드로이드 기기 - 512x512: PWA 로딩시 스플래시 스크린용

• HTML5

- W3C기준 - 자식,자손 요소 들여쓰기 - 디바이스별 환경 극복을 위한 meta[name: viewport] 설정 - css vendor의 손쉬운 적용을 위해 prefixfree.js 연결 - header/ main/ aside/ footer를 이용하여 영역 분리 - img 태그 사용시 alt값 필수 작성 - 외부 링크는 a 태그 사용
: 빈 링크의 href의 값은 #로 통일
- 식별자명 규칙
- id값을 중복 사용하지 않는다 - 첫글자는 알파벳으로 시작 -알파벳, 숫자, 언더바(_)를 조합하여 부여

• CSS3

- 첫줄에 인코딩 선언 @charset "UTF-8"; - 웹페이지 공통 css속성은 common폴더 속 style.css에 작성 - 주석을 이용하여 영역별 css 구분 - 모든 속성선언 끝에 세미콜론(;) 작성

• 파일명

- 웹페이지의 첫 페이지는 index.html로 통일 - 파일명 알파벳, 숫자, 언더바(_)를 조합하여 부여 - 명확하고 간결하게 네이밍한다.

• 폴더 구조

- common:
- css : 페이지에 공통으로 연결된 style.css및 font.css가 담겨있는 폴더 - favicon: 6개의 이미지와 1개의 JSON 파일이 담겨있는 폴더 - fonts: 웹폰트를 제외한 폰트 파일이 담겨있는 폴더 - images: 웹에서 사용된 모든 이미지가 담겨있는 폴더 - scripts: 페이지에 공통으로 연결된.js 파일이 담겨있는 폴더
